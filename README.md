# AssetChain

Веб-система количественного учета имущества, складских остатков и перемещений с
Telegram-подтверждениями.

Полное руководство по ролям, страницам, кнопкам, поступлениям, перемещениям и
связям базы данных: [docs/FULL_SYSTEM_GUIDE_RU.md](docs/FULL_SYSTEM_GUIDE_RU.md).

## Быстрый запуск через Docker

1. Скопируйте `.env.example` в `.env` и заполните секреты.
2. Запустите:

   ```bash
   docker compose up -d --build
   docker compose exec web python manage.py createsuperuser
   ```

3. Откройте `http://SERVER:8000/`.

Пользователи, склады, ответственные и согласующие настраиваются в разделе
`Управление`. Для каждого активного склада обязательно назначается согласующий:
любое перемещение сначала получает статус «На согласовании».

## Telegram

Создайте бота через BotFather и задайте в `.env`:

```env
TELEGRAM_BOT_TOKEN=...
TELEGRAM_BOT_USERNAME=AssetChainBot
```

После создания пользователя откройте его карточку в админ-панели. Там появится
персональная ссылка привязки Telegram. Компонент `bot` в Docker Compose запускает
long polling автоматически.

## Импорт Excel

Загрузку выполняет главный администратор через раздел `Импорт`. Нужны три листа:

- `Номенклатура`: Наименование, Артикул, Штрихкод; необязательно Категория, Описание.
- `Склады`: Наименование.
- `Остатки`: Склад, Артикул, Количество.

Импортированные артикулы и штрихкоды сохраняются как есть. Новые коды
генерируются с проверкой по всей базе, включая импортированные записи. Импорт
атомарный: ошибка в любой строке отменяет все изменения файла.

## Локальная разработка

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```

Проверка:

```bash
python manage.py test
python manage.py check
```
